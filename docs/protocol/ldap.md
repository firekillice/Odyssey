## LDAP（Light Directory Access Portocol）
* 轻量目录访问协议
* 访问的信息存储在一个目录下，是一种nosql的树形数据
* 其schema天生与公司的组织架构一致
* 其实本质上讲，是对某种组织信息的管理
* 数据也可以存储在其他的数据库中，比如mysql、mongodb，只要符合LDAP的查询协议规范即可
* 有点类似游戏管理平台中的权限管理
* 目前更多用于公司内部的业务授权认证


## 授权的几种级别
* 跨域: OAuth2.0
* 公司内部: LDAP 
* 网站内部的用户权限管理、OS的权限管理
* TLS的信任传递
